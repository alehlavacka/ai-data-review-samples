# Sample Annotation Guidelines

This document outlines simple guidelines for text annotation tasks.

## Objective

Ensure consistent labeling across similar types of content.

## General Rules

1. Follow the predefined tag set strictly.
2. Do not infer information that is not explicitly stated.
3. If the text is ambiguous, flag it for review.
4. Maintain consistency across similar examples.

## Tone Classification

- Neutral: Informational, no emotional language.
- Positive: Contains supportive or optimistic language.
- Negative: Contains criticism or strong negative sentiment.

## Common Mistakes to Avoid

- Over-labeling
- Making assumptions
- Ignoring edge cases
- Inconsistent tag application

## Quality Control Checklist

- Are tags applied consistently?
- Does the label match the content exactly?
- Are there unclear cases that need review?

This sample demonstrates structured thinking and attention to detail in annotation workflows.
